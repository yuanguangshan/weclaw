# WeClaw 项目架构解读

本项目是一个**微信 AI Agent 桥接服务**，将微信消息路由到多种 AI Agent（Claude、Codex、GPT-4 等）。

---

## 📁 文件夹概览

| 文件夹 | 代码量 | 作用 |
|--------|--------|------|
| [`agent/`](agent.md) | ~2000 行 | AI Agent 抽象层（ACP/CLI/HTTP） |
| [`api/`](api.md) | ~1050 行 | HTTP API 服务 + 管理后台 |
| [`cmd/`](cmd.md) | ~900 行 | CLI 命令行工具 |
| [`config/`](config.md) | ~560 行 | 配置管理 + Agent 自动检测 |
| [`messaging/`](messaging.md) | ~5100 行 | 消息路由 + 业务逻辑核心 |
| [`hub/`](hub-ilink-web.md) | ~650 行 | Agent 共享文件中心 |
| [`ilink/`](hub-ilink-web.md) | ~660 行 | 微信 iLink API 客户端 |
| [`web/`](hub-ilink-web.md) | ~1330 行 | 管理后台前端（HTML 嵌入） |

## 🚀 快速开始

```bash
# 安装
go install github.com/fastclaw-ai/weclaw@latest

# 启动（自动检测已安装的 Agent）
weclaw start

# 扫码登录
weclaw login

# 发送消息
weclaw send --to "user_id" --text "Hello"

# 查看状态
weclaw status
```

## 🎯 核心特性

- **多 Agent 支持**：Claude、Codex、Gemini、Cursor、Kimi 等 13+ 种
- **零配置启动**：自动检测系统中已安装的 AI 工具
- **微信集成**：通过 iLink 协议接收/发送微信消息
- **Hub 协作**：多 Agent 文件共享与上下文传递
- **链式协作**：`/hub pipe` 实现多 Agent 流水线处理
- **辩论功能**：`/debate` 多 Agent 多轮辩论
- **定时任务**：`/todo` 待办事项 + `/timer` 定时器
- **在线更新**：`weclaw update` 自动从 GitHub 更新
