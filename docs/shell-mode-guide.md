# WeClaw 命令行模式使用指南

> 在微信中使用安全的 Shell 命令查看和管理远程文件

## 功能概述

WeClaw 命令行模式允许你在微信中直接执行安全的 Shell 命令，方便远程查看日志、管理文件和监控系统状态。所有命令都经过安全白名单过滤，确保系统安全。

---

## 快速开始

### 进入命令行模式

```
/sh
```

进入后会显示：

```
--- 当前为命令行模式 (/q 退出) ---
当前目录: /root/.weclaw/workspace/weclaw
提示: 直接输入命令即可，无需 /sh 前缀
支持 cd 切换目录，目录会持久化保存
```

### 退出命令行模式

```
/q
```

---

## 使用方式

### 方式一：命令行模式（推荐）

适合需要连续执行多个命令的场景。

```
/sh          # 进入命令行模式
ls -la       # 直接输入命令，无需前缀
cd /tmp      # 切换目录（会持久化）
pwd          # 查看当前目录
cat log.txt  # 查看文件
/q           # 退出模式
```

### 方式二：单次执行

适合执行单个命令，不想进入模式。

```
/sh ls -la
/sh pwd
/sh cat /tmp/log.txt
```

## 支持的命令

### 文件浏览

| 命令 | 说明 | 示例 |
|------|------|------|
| `ls` | 列出文件 | `ls -la`, `ls *.txt` |
| `pwd` | 显示当前目录 | `pwd` |
| `cd` | 切换目录（持久化） | `cd /tmp`, `cd ..` |
| `tree` | 目录树 | `tree -L 2` |
| `file` | 文件类型 | `file app.go` |

### 文件内容查看

| 命令 | 说明 | 示例 |
|------|------|------|
| `cat` | 查看文件内容 | `cat log.txt` |
| `head` | 文件头部 | `head -n 20 log.txt` |
| `tail` | 文件尾部 | `tail -n 100 log.txt` |
| `nl` | 添加行号 | `nl file.txt` |

### 搜索与路径

| 命令 | 说明 | 示例 |
|------|------|------|
| `grep` | 搜索文本 | `grep "error" log.txt` |
| `find` | 查找文件 | `find . -name "*.log"` |
| `which` | 定位命令路径 | `which go` |
| `realpath` | 获取绝对路径 | `realpath ./file.txt` |
| `readlink` | 读取符号链接 | `readlink /usr/bin/python` |
| `basename` | 获取文件名 | `basename /usr/local/bin/go` |
| `dirname` | 获取目录路径 | `dirname /usr/local/bin/go` |

### 文本处理

| 命令 | 说明 | 示例 |
|------|------|------|
| `echo` | 输出文本 | `echo $HOME` |
| `wc` | 统计行数/字数 | `wc -l file.txt` |
| `sort` | 排序 | `sort names.txt` |
| `uniq` | 去重 | `sort file.txt \| uniq` |
| `cut` | 切割字段 | `cut -d: -f1 /etc/passwd` |
| `awk` | 文本处理 | `awk '{print $1}' log.txt` |
| `sed` | 文本替换 | `sed 's/old/new/g' file.txt` |
| `tr` | 字符替换/删除 | `tr 'a-z' 'A-Z' < file.txt` |
| `xargs` | 参数传递 | 需配合管道使用 |

### 系统信息

| 命令 | 说明 | 示例 |
|------|------|------|
| `du` | 磁盘使用 | `du -sh *` |
| `df` | 磁盘空间 | `df -h` |
| `date` | 系统时间 | `date` |
| `stat` | 文件状态 | `stat file.txt` |

### 不支持的交互式命令

| 命令 | 原因 | 替代方案 |
|------|------|----------|
| `htop` / `top` | 交互式界面 | 使用 `df -h`, `du -sh` 查看资源 |
| `vim` / `nano` | 需要持续输入 | 使用 `cat` / `head` / `tail` 查看内容 |
| `less` / `more` | 交互式分页 | 使用 `head -n 100` / `tail -n 100` |
| `alias` | 可绕过安全限制 | 使用完整命令或快捷指令 |

### 快捷指令

| 快捷指令 | 等同于 | 说明 |
|----------|--------|------|
| `ll` | `ls -lh` | 详细列表 |
| `..` | `cd ..` | 返回上级目录 |
| `...` | `cd ../..` | 返回上两级目录 |

## 安全特性

### 多层安全防护

1. **命令白名单**: 只允许安全的命令
2. **操作符过滤**: 阻止 `|`, `>`, `&&`, `;` 等危险操作符
3. **路径沙盒**: 可配置限制在指定目录内操作
4. **软链接防护**: 自动解析符号链接，防止逃逸

### 被阻止的操作示例

```
❌ cat /etc/passwd | grep root  # | 被阻止
❌ echo "test" > file.txt       # > 被阻止
❌ rm -rf /                     # rm 不在白名单
❌ cat file.txt && ls           # && 被阻止
```

## 使用技巧

### 1. cd 后自动 ls

切换目录后会自动显示目录内容：

```
cd /var/log
# 输出：
✅ 已切换到: /var/log

...文件列表...
```

### 2. 多列显示优化

`ls` 命令自动优化为多列显示，目录优先排在前面：

```
ls
# 输出：
Documents/  Downloads/  Pictures/  file.txt  image.png
```

### 3. 输出格式化

所有命令输出都用代码块包裹，微信会以等宽字体显示：

```
```
文件内容
保持原有格式
```
```

## 常见场景示例

### 查看应用日志

```
/sh           # 进入命令行模式
cd /var/log   # 切换到日志目录
tail -f app.log | grep ERROR  # ❌ 不支持管道
tail -n 100 app.log           # ✅ 查看最后 100 行
```

### 检查磁盘空间

```
/sh df -h    # 单次执行
```

### 搜索配置文件

```
/sh find /etc -name "*.conf"  # 查找所有配置文件
```

### 统计代码行数

```
/sh find . -name "*.go" | xargs wc -l  # ❌ 不支持管道
/sh wc -l *.go                        # ✅ 统计当前目录 go 文件
```

## 限制说明

### 微信平台限制

1. **消息长度**: 输出超过 4000 字符会自动截断
2. **等宽字体**: 微信对代码块的支持有限，多列对齐可能不完美
3. **交互限制**: 不支持需要用户输入的交互式命令

### 功能限制

1. 不支持管道符和重定向
2. 不支持后台任务 (`&`)
3. 不支持命令替换 (`$()` 或 `` ` ``)
4. 不在白名单的命令无法执行
5. 不支持交互式命令（如 `vim`, `nano`, `htop`, `top`）
6. 不支持 `alias` 和环境变量修改命令

## 故障排查

### 命令执行失败

如果显示 `❌ 命令执行失败`：

1. 检查命令拼写是否正确
2. 确认命令在白名单中
3. 检查文件路径是否存在
4. 确认是否有权限访问

### 输出被截断

如果显示 `... (输出已截断)`：

1. 使用 `head` 或 `tail` 查看部分内容
2. 使用 `grep` 过滤需要的内容
3. 分批查看大文件

### 路径超出范围

如果显示 `❌ 路径超出允许范围`：

1. 管理员可能配置了路径沙盒
2. 只能在允许的目录内操作
3. 联系管理员调整配置
